May 16th, 2014 by Robin T. Miller

	There's a script named makedt, which is what I use for building the various dt's,
for various OS's. Please Note: Depending on the OS version, additional build parameters
may be defined (see makedt for details).

	The proper way to build dt is to create a subdirectory, which makedt does, then cd there
to do a build similar to this simple script (see below).

	The Makefile for each OS being built will use the OS variable to create a symbolic link to
the appropriate SCSI library, if one exists for that OS (none for FreeBAD or MacOS at this time).
This, of course, is new since this is the first dt release with the SCSI library included.

	Hope this helps!

Cheers,
Robin
